1まず3つの問題を分解する

本記事では既報の 国内MaaS大戦（「二つの戦線」について既に説明した）や 中国Token海外展開（「OpenRouter 61%のシェア」について既に説明した）を繰り返さない。今回、以下の3つの新たな問題だけに答える：

  1. 商業クローズドループ — Token販売による収入が、計算リソース + 研究開発 + 販売コストをカバーできるか？どの企業が成立させているのか？
  2. ポジティブサイクル — クローズドループが成立した後、ビジネスが自己強化できるか（モデルが良くなる → 顧客が増える → データが還流する → モデルがさらに良くなる）？

3MaaS販売パラダイムは学ぶ価値があるか

3.1 中国のMaaS販売パラダイムの核心

ある大手総合クラウドBが2025-2026年に推進するMaaS販売パラダイムのアップグレード：

従来のクラウド販売（リソース販売） → MaaS販売（トークン販売） → エージェント販売（能力/シナリオ/結果の販売）

コアアクション：

  1. 複数の新しいMaaS販売チームを設立し、既存の直販チームと協力して受注を行う
  2. コアKPI：利用シーンの発掘 + 企業のエージェント活用促進 + トークン消費量の増加
  3. 付随製品：業界別エージェントテンプレート（Eコマースカスタマーサポート / 金融リスク管理 / 行政問合せ / 工業文書）
  4. 価格モデル：年間契約枠 + トークンパッケージ + 成果報酬（GMV連動 / 節約工数連動）

ある大手動画クラウドAのより過激なバリエーション：モデル甲は直接ホワイトリスト + 年間枠方式を採用。春節のピーク時には10時間待ち → 「最高入札額 + 最も業務の深い」顧客を強制的に選別。「フルスペック版」の安定供給 → GPUリソースを年間契約顧客に優先提供。

3.3 「形 vs. 神」の分解

学べる形（営業組織 / プロセス / ツール）

  • ✅ MaaS専任販売チームの設立、既存直販との協業
  • ✅ KPI設計：シナリオ数 + トークン消費量 + ビジネス成果
  • ✅ 業界別エージェントテンプレートライブラリ
  • ✅ 年間枠 + トークンパッケージ + 成果報酬型価格
  • ✅ 付随するホワイトペーパー + ROI試算ツール

学べない神（根幹的な優位性）

  • ❌ グループ内シナリオ養殖 — ある大手動画クラウドAは抖音ショートドラマ（年220～300億）、ある大手総合クラウドBは淘宝カスタマーサービス（数百万の事業者）、ある大手総合クラウドCは公式アカウントを持つ。
  • ❌ グループ内GPU/電力スケジューリング — 「自社のGPU遊休時間」をMaaS事業に割引提供できる
  • ❌ 国産GPU代替 — 国産Aカードの「使えれば良し」の姿勢で、H100/H200の禁輸後も中国大手は推論能力を維持可能
  • ❌ データコンプライアンスの優位性 — 国内大手顧客（金融/政府・企業/中央企業）は自然と国内クラウドを選択する

3.4 誰にとって学ぶに値するか

企業タイプ学ぶ価値があること学べないこと
産業シーンを持つクラウド事業者全面的に学べる（形 + KPI調整）自社の魚養殖には時間が必要
純AIスタートアップ形は学べる（営業組織/価格モデル）魚養殖/計算能力/データコンプライアンスは学べない
従来業界のISV/SIMaaS販売KPI体系を学ぶ自社開発モデル不足/計算能力不足
海外大手「年間枠 + トークンパッケージ」価格を学ぶ中国大手の「グループ内魚養殖」は学べない

4価格が10分の1でも機能する5つのメカニズム

これは本稿で最もコアな節である。中国のAPI価格は長期的に海外の10分の1以下だが、市場は崩壊しておらず、補助金によるものではなく、5つの基層メカニズムが同時にボリュームを引き上げ、3つの中国特有の要素が重なっているからである。

メカニズム1：ロングコンテキスト税

現象：2025年～2026年、主流モデルのコンテキストウィンドウは8Kから128K / 1M / 10Mへと飛躍的に拡大する。

  • 1回のRAGリクエストに50万～100万文字のコンテキストを詰め込む
  • 1回のAPI呼び出しあたりのトークン数は約1Kから約500Kへ（500倍）
  • 顧客は「価格は変わっていない」と感じる（API単価は5～10分の1に低下）、しかし実際の消費金額は50倍に増加

トークン単価は90%下落、しかし1回あたりの金額は75倍に → 顧客の総請求額は逆に7.5倍に増加。モデル事業者（1回あたりの課金額増）＋上位アプリ（ユーザー体験向上）→ ウィンウィン。

メカニズム2：出力インフレ

現象：モデルは「安全で冗長な出力」を好む。本来50字で説明できる内容を、AIは500字書く。

  • 初期のGPT-4：平均出力200トークン/リクエスト
  • 2026年第1四半期の主流モデル：平均出力800トークン/リクエスト
  • 出力トークンは4倍に増加（入力は変わらずとも）

出力トークンの価格は通常、入力の3～5倍である（特にコーディングや動画生成）。顧客は「出力トークン」に対してより多くの料金を支払う → 事業者の収益増加；しかしGPUの実計算量は4倍には増えていない（キャッシュ / 投機的デコード / 量子化）→ 限界費用の伸びは収益の伸びを大きく下回る。→ 事業者にとって：粗利率が拡大

メカニズム3：スケジューリング割引

1トークンあたりの限界費用は4つの技術的手段を通じて継続的に低下する：

技術1トークンあたりコスト低下
キャッシュ（Cache）キャッシュヒット → 90%低下
量子化（Quantization）FP16 → INT8 / INT4 → 50～75%低下
MoEアーキテクチャアクティブパラメータを3～5%に圧縮 → 急激な低下
投機的デコード推論時間を30～50%短縮

データ（オープンソース先導チームRの公開技術報告書）：

  • 1トークンあたり推論コスト 2024年第4四半期：$0.0014
  • 1トークンあたり推論コスト 2026年第1四半期：$0.0002（86%低下）
  • API販売価格 2024年第4四半期：$0.27/百万トークン
  • API販売価格 2026年第1四半期：$0.014/百万トークン（95%低下）
  • 販売価格の下落がコストより速い → 粗利率はむしろ拡大

メカニズム4：予算の内製化

AI呼び出しは「研究開発プロジェクト支出」から「運営の定常支出」へと変化する。

  • 2024年：CFOの視点＝「これは研究開発部の新しいおもちゃ」
  • 2026年：CFOの視点＝「これは光熱費＋SaaSサブスクリプション＋クラウドリソース」

事業者にとっての意味

  • 顧客内部の承認経路が短縮（IT部門の決定権が拡大）
  • 更新率が向上（60～80%業界基準）
  • ARPUが向上（CFOが自発的に予算上限を引き上げる）

データ：2024年の企業AI予算中央値 $50K/年 → 2026年 $500K/年（10倍増）。主なドライバー：「パイロット」→「スケール化」→「固定支出」の三段跳び。

メカニズム5：コンプライアンスプレミアム

監査可能 / 出所記録付き / プライベートデプロイのトークンは依然として30～100%のプレミアムがつく。

業界プレミアム倍率
金融業界の監査トークン1.3～2.0倍標準価格
政府・企業のプライベートデプロイ1.5～3.0倍標準価格
医療 / 法律の追跡可能トークン1.2～1.5倍標準価格

標準APIの価格競争は激しい（粗利率30～50%）、コンプライアンス / プライベートデプロイ / 業界カスタマイズのトークンは依然として高粗利（60～80%）。MaaS販売の中核的利益源はコンプライアンスプレミアムであり、標準APIではない。

  • 1トークンあたり粗利率は海外と同等かそれ以上（構造的低コスト＋コンプライアンスプレミアムの強さ）

戦略1：シナリオ養魚 → トークン収穫

ある大手総合クラウドBの戦略：

  1. 淘宝 / 1688のマーチャントカスタマーサービスシナリオ → モデル乙の能力を検証

オープンソース先導チームRの戦略：

  1. V2 / V3 すべてMITライセンスでオープンソース化
  2. 価格を極限まで低く設定（$0.014/百万キャッシュヒット）
  3. OpenRouterで28%のシェアを獲得 → 世界中の開発者が採用
  4. 海外からの呼び出し量 → 国内GPU使用率が向上
  5. 国内の価格設定は低いが、グローバルな呼び出しがトレーニング＋推論＋研究開発の固定費を分散させている

6反対意見 / リスクポイント

6.1 価格が1/10でも機能する3つの懸念

懸念1：補助金合戦はいつまで続くのか？

現在、中国のAPI価格 = コスト + わずかな利潤（真の市場清算価格ではない）。もしある日、業界の淘汰が進み（2～3社の寡占企業が残る）、価格が5～10倍に跳ね返る可能性がある。類比：中国クラウド市場の2018～2020年の価格戦争 → 2021～2022年に30～50%の値上げ。

懸念2：オープンソースのフライホイールは持続可能か？

オープンソースをリードするチームR / モデル乙は現在「ボランティア精神 + 国家戦略支援」で動いている。もし政策支援が弱まるか、チームに商業化の圧力が高まれば → オープンソースライセンスが引き締められる可能性がある。類比：Red Hat / MongoDB / Elasticは皆「オープンソース → 商業化による厳格化」の道を歩んできた。

懸念3：コンプライアンス・プレミアムの持続可能性

現在のコンプライアンス / プライベート化プレミアムの30～100%は「国産代替 + データセキュリティ」政策に基づいている。もし海外モデルが再び中国参入を許可されれば → コンプライアンス・プレミアムは圧縮されるだろう。

6.2 クローズドループの3つの脆弱な点

脆弱点1：単一モデルのウィンドウ期間 — モデル甲は現在、短尺ドラマでの浸透率95% → 業界飽和；海外動画モデルが第3世代で追い上げ → そのモートはどれだけ持続するか？判断：動画モデルのウィンドウ期間は18～24ヶ月であり、次世代モデルへの継続的な投資が必要。

脆弱点2：コーディングモデルの同質化 — モデル丙は現在コーディング分野でトップだが、海外モデル + Cursor / Devin が競合している；海外モデルが再び参入許可されれば、コーディングのトップは交代する可能性がある。判断：コーディングのウィンドウ期間は12～18ヶ月であり、開発者エコシステムとの結びつきが必要。

脆弱点3：汎用LLMの価格戦争の泥沼 — 国産オープンソースの7B / 70Bの価格は既に0.5～2元/百万トークンまで低下しており、さらなる値下げ余地は限られている（既に限界費用に近い）。判断：汎用LLM市場は2年以内に3～5社の寡占に収束し、価格戦争は終了する。

7結論：ビジネス閉ループが成立する条件

7.1 成立する4つの条件（同時に満たす）

  • ✅ 条件 1：モデルがある垂直分野でトップ3に入る（上位3位）
  • ✅ 条件 2：顧客業務との深い結びつき（移行コスト > 切り替え利益）
  • ✅ 条件 3：トークン収入 > 計算力 + 研究開発 + 販売コスト（粗利益率 > 30%）
  • ✅ 条件 4：「自家鱼塘」、「オープンソースフライホイール」、「コンプライアンスプレミアム」のいずれか一つを持つ

7.3 読者への判断

中国MaaSプレイヤー向け

  • 汎用LLMレースは18ヶ月以内に淘汰される
  • 垂直分野（動画 / Coding / ショートドラマ / 業界Agent）が2026-2028年に実際に利益を生む主戦場となる
  • 「自家鱼塘」こそが究極のモートであり、グループ内シナリオで育成→トークン収益化の道筋は丸ごと学ぶ価値がある

顧客 / 投資家向け

  • 「値上げしても量が減らない」現象はトップAIモデルで成立している（CodingリーダーXで実証済み）
  • 1/10価格は2〜3年持続し、その後5〜10倍に跳ね返る（業界淘汰）
  • オープンソースフライホイール = グローバルコールが国内コストを薄め、オープンソースリーダーチームR / モデル乙 が構造的勝者

海外読者向け

  • 中国MaaS販売パラダイムは「形は学べるが神髄は学べない」
  • 真のモートはグループ内シナリオでの育成であり、これは中国インターネット大手の独自資産だ
  • 海外は「年間枠 + トークンパッケージ + 成果分配」の価格モデルを学べるが、「自家鱼塘」は学べない
