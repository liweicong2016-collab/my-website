EN Summary: This report dissects whether China's MaaS (Model-as-a-Service) business has a closed-loop / positive-cycle commercial model. Three new angles: (1) unit economics of single token — 70-93% gross margin for vertical winners; (2) MaaS sales playbook — copy the form, not the essence; (3) 5 mechanisms sustaining 1/10 pricing: long-context tax, output inflation, scheduling discount, budget internalization, compliance premium + 3 China-specific overlays.

1先把 3 个问题拆清楚

这一篇不重复 国内 MaaS 大战（已讲"两条战线"）和 中国 Token 出海（已讲"OpenRouter 61% 份额"）。这次只回答 3 个新问题：

  1. 商业闭环 — 卖 Token 收入能不能覆盖算力 + 研发 + 销售成本？哪些公司跑通了？
  2. 正循环 — 闭环跑通后，业务能不能自我强化（模型越好 → 客户越多 → 数据回流 → 模型更好）？
  3. 价格 1/10 还能运转 — 中国 API 价格长期低于海外 10 倍，市场没崩。靠的是哪 5 个底层机制？

3MaaS 销售范式是否值得学

3.1 中国 MaaS 销售范式的内核

某头部综合云 B 2025-2026 推动的 MaaS 销售范式升级：

传统云销售（卖资源） → MaaS 销售（卖 Token） → Agent 销售（卖能力 / 卖场景 / 卖结果）

核心动作：

  1. 成立数个新 MaaS 销售团队，与原直销团队合作打单
  2. 核心 KPI：挖掘使用场景 + 推动企业用 Agent + 增加 Token 消耗
  3. 配套产品：行业 Agent 模板（电商客服 / 金融风控 / 政务问答 / 工业文档）
  4. 定价模型：年框 + Token 包 + 效果分成（按 GMV / 按节省工时）

某头部视频云 A 的变体（更激进）：模型甲直接白名单 + 年框模式；春节高峰期排队 10 小时 → 强行筛选"出价最高 + 业务最深"的客户；"满血版"稳定供应 → 把 GPU 资源优先给年框客户。

4价格 1/10 还能运转的 5 个机制

这是本文最硬核的一节。中国 API 价格长期低于海外 10 倍，市场没崩，不是靠补贴，是 5 个底层机制同时拉量 + 3 个中国特色叠加。

机制 1：长上下文税

现象：2025-2026 主流模型上下文窗口从 8K 跃迁到 128K / 1M / 10M。

  • 一次 RAG 请求塞 50-100 万字上下文
  • 单次 API 调用 token 数从 ~1K 涨到 ~500K（500 倍）
  • 客户感知"价格没变"（API 单价降了 5-10 倍），但实际消费金额涨 50 倍

token 单价跌 90%，但单次金额涨 75 倍 → 客户总账单反而涨 7.5 倍。模型厂商（单次收费更多）+ 上层应用（用户体验更好）→ 双赢。

机制 2：输出通胀

现象：模型倾向"安全冗长输出"。一段本可以 50 字说清的话，AI 会写 500 字。

  • 早期 GPT-4：平均输出 200 tokens / 请求
  • 2026 Q1 主流模型：平均输出 800 tokens / 请求
  • 输出 token 涨 4 倍（即便输入没变）

输出 token 价格通常是输入的 3-5 倍（Coding / 视频生成尤其）。客户按"输出 token"付费更多 → 厂商收入增长；但 GPU 实际计算量没有 4 倍增长（缓存 / 投机解码 / 量化）→ 边际成本增速远低于收入增速。→ 对厂商：毛利率扩大

机制 3：调度折价

单 token 边际成本通过 4 个技术手段持续下降：

技术单 token 成本下降
缓存（Cache）命中缓存 → 降 90%
量化（Quantization）FP16 → INT8 / INT4 → 降 50-75%
MoE 架构激活参数压到 3-5% → 断崖下降
投机解码推理时间降 30-50%

数据（开源领军团队 R 公开技术报告）：

  • 单 token 推理成本 2024 Q4：$0.0014
  • 单 token 推理成本 2026 Q1：$0.0002（降 86%）
  • API 售价 2024 Q4：$0.27/百万
  • API 售价 2026 Q1：$0.014/百万（降 95%）
  • 卖价跌得比成本快 → 毛利率反而扩

机制 4：预算内化

AI 调用从"研发项目支出"变成"运营常规支出"。

  • 2024 年：CFO 视角 = "这是研发部的新玩具"
  • 2026 年：CFO 视角 = "这是水电费 + SaaS 订阅 + 云资源"

对厂商的含义

  • 客户内部审批路径缩短（IT 部门决定权增大）
  • 续约率提升（60-80% 行业基准）
  • ARPU 提升（CFO 主动加预算上限）

数据：2024 年企业 AI 预算中位数 $50K/年 → 2026 年 $500K/年（涨 10 倍）。主要驱动：从"试点"→"规模化"→"固定支出"三级跳。

机制 5：合规溢价

可审计 / 来源记录 / 私有化部署的 token 仍能溢价 30-100%。

行业溢价倍数
金融行业审计 token1.3-2.0x 标准价
政企客户私有化部署1.5-3.0x 标准价
医疗 / 法律可溯源 token1.2-1.5x 标准价

标准 API 价格战激烈（毛利率 30-50%），合规 / 私有化 / 行业定制 token 仍是高毛利（60-80%）。MaaS 销售的核心利润来源就是合规溢价，不是标准 API。

6反方观点 / 风险点

6.1 价格 1/10 还能运转的 3 个隐忧

隐忧 1：补贴战能持续多久？

当前中国 API 价格 = 成本 + 微利（不是真正的市场出清价）。如果某天行业出清（剩 2-3 家寡头），价格可能反弹 5-10 倍。类比：中国云市场 2018-2020 价格战 → 2021-2022 涨价 30-50%。

隐忧 2：开源飞轮能不能持续？

开源领军团队 R / 模型乙 当前是"用爱发电 + 国家战略支持"。如果政策支持减弱，或者团队商业化压力增大 → 开源协议可能收紧。类比：Red Hat / MongoDB / Elastic 都走过"开源 → 商业化收紧"的路径。

隐忧 3：合规溢价的可持续性

当前合规 / 私有化溢价的 30-100% 是基于"国产替代 + 数据安全"政策。如果海外模型重新获准进入中国 → 合规溢价会被压缩。

6.2 闭环的 3 个脆弱点

脆弱点 1：单点模型的窗口期 — 模型甲当前短剧渗透率 95% → 行业饱和；海外视频模型 3 代跟进 → 护城河能持续多久？判断：视频模型窗口期 18-24 个月，需持续投入下一代模型。

脆弱点 2：Coding 模型的同质化 — 模型丙当前 Coding 头部，但海外模型 + Cursor / Devin 都在抢；海外模型一旦重新获准，Coding 头部位置可能轮换。判断：Coding 窗口期 12-18 个月，需绑定开发者生态。

脆弱点 3：通用 LLM 的价格战泥潭 — 国产开源 7B / 70B 价格已经低到 0.5-2 元/百万 tokens；继续降价空间有限（已经接近边际成本）。判断：通用 LLM 赛道 2 年内会剩 3-5 家寡头，价格战结束。
